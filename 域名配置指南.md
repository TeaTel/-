# 域名 c2cmarket.store 配置指南

## 概述
本指南将帮助您将域名 `c2cmarket.store` 绑定到校园二手交易平台项目。

## 域名架构设计

```
c2cmarket.store (主域名)
├── 前端访问: https://c2cmarket.store
├── www子域名: https://www.c2cmarket.store
└── API子域名: https://api.c2cmarket.store
```

## 步骤1: 在Render配置自定义域名

### 1.1 部署项目到Render
1. 推送代码到GitHub
2. 在Render控制台创建新项目
3. 选择"Blueprint"部署方式
4. Render会自动读取 `render.yaml` 文件

### 1.2 获取Render分配的域名
部署完成后，您将获得：
- 前端: `campus-marketplace.onrender.com`
- 后端API: `campus-marketplace-api.onrender.com`

### 1.3 在Render控制台添加自定义域名
1. 登录Render控制台: https://dashboard.render.com
2. 选择 `campus-marketplace` 服务
3. 进入"Settings" → "Custom Domains"
4. 添加以下域名:
   - `c2cmarket.store`
   - `www.c2cmarket.store`
5. 选择 `campus-marketplace-api` 服务
6. 添加API域名:
   - `api.c2cmarket.store`

### 1.4 获取DNS配置信息
添加域名后，Render会显示DNS配置信息：
```
类型: CNAME
名称: @ (或留空)
值: campus-marketplace.onrender.com
TTL: 3600

类型: CNAME  
名称: www
值: campus-marketplace.onrender.com
TTL: 3600

类型: CNAME
名称: api
值: campus-marketplace-api.onrender.com
TTL: 3600
```

## 步骤2: 在阿里云配置DNS解析

### 2.1 登录阿里云控制台
1. 访问: https://dns.console.aliyun.com
2. 选择您的域名 `c2cmarket.store`

### 2.2 添加DNS解析记录

#### 记录1: 主域名解析 (A记录或CNAME)
```
记录类型: CNAME
主机记录: @
记录值: campus-marketplace.onrender.com
TTL: 600秒 (10分钟)
```

#### 记录2: www子域名
```
记录类型: CNAME
主机记录: www
记录值: campus-marketplace.onrender.com
TTL: 600秒
```

#### 记录3: api子域名
```
记录类型: CNAME
主机记录: api
记录值: campus-marketplace-api.onrender.com
TTL: 600秒
```

## 步骤3: SSL证书配置

### 3.1 Render自动SSL
- Render会自动为所有自定义域名申请免费的Let's Encrypt SSL证书
- 证书会自动续期
- 需要等待DNS解析生效后（通常几分钟到几小时）

### 3.2 验证SSL证书
部署完成后，访问以下地址验证HTTPS:
- https://c2cmarket.store
- https://www.c2cmarket.store  
- https://api.c2cmarket.store

## 步骤5: 更新应用配置

### 5.1 前端环境变量
已更新 `.env.production` 文件:
```env
VITE_API_BASE_URL=https://api.c2cmarket.store
```

### 5.2 后端CORS配置
已更新 `render.yaml` 中的CORS配置:
```yaml
ALLOWED_ORIGINS: "https://c2cmarket.store,https://www.c2cmarket.store,http://localhost:3000"
```

## 常见问题解决

### 问题1: DNS解析不生效
**症状**: 域名无法访问或显示"无法连接"
**解决**:
1. 检查阿里云DNS配置是否正确
2. 等待DNS传播（最多24-48小时）
3. 使用 `nslookup` 命令测试解析
4. 清除本地DNS缓存

### 问题2: SSL证书错误
**症状**: 浏览器显示"不安全"或证书错误
**解决**:
1. 等待Render自动申请证书（最多2小时）
2. 在Render控制台检查SSL状态
3. 确保DNS解析已生效

### 问题3: 前端无法连接API
**症状**: 前端页面显示API连接错误
**解决**:
1. 检查 `VITE_API_BASE_URL` 配置
2. 验证 `api.c2cmarket.store` 是否解析正确
3. 检查后端CORS配置
4. 查看浏览器开发者控制台错误信息

### 问题4: 子域名无法访问
**症状**: `www.c2cmarket.store` 或 `api.c2cmarket.store` 无法访问
**解决**:
1. 检查阿里云DNS子域名配置
2. 验证Render中已添加对应域名
3. 测试每个子域名的DNS解析

## 域名管理最佳实践

### 1. DNS记录管理
- 使用较短的TTL（600秒）便于快速更新
- 定期备份DNS配置
- 记录所有DNS变更

### 2. SSL证书监控
- Render自动管理SSL证书
- 定期检查证书状态
- 设置证书过期提醒

### 3. 域名续费
- 设置域名自动续费
- 提前30天续费避免过期
- 保存域名注册商联系信息

### 4. 安全考虑
- 启用DNSSEC（如果支持）
- 使用强域名注册商密码
- 开启二次验证
