## 技术栈

### 前端技术栈
- **框架**: Vue 3.4.0 (Composition API)
- **构建工具**: Vite 5.0.0
- **路由**: Vue Router 4.2.5
- **HTTP客户端**: Axios 1.6.0
- **状态管理**: Vue Composition API
- **UI组件**: 自定义组件 + 原生HTML/CSS
- **Node版本**: >=18.0.0

### 后端技术栈
- **框架**: Spring Boot 3.x
- **数据库**: MySQL + MyBatis
- **缓存**: Redis (已禁用，可选项)
- **认证**: JWT (JSON Web Token)
- **API文档**: Swagger/OpenAPI
- **构建工具**: Maven
- **Java版本**: 21
- **监控**: Spring Boot Actuator

## 环境变量配置

### 前端环境变量 (Vercel)
| 变量名 | 必需 | 默认值 | 说明 |
|--------|------|--------|------|
| `VITE_API_BASE_URL` | ✅ | `http://localhost:8080` | 后端API地址 |
| `VITE_APP_NAME` | ❌ | `校园二手交易平台` | 应用名称 |
| `VITE_APP_VERSION` | ❌ | `1.0.0` | 应用版本 |
| `VITE_DEBUG` | ❌ | `false` | 调试模式 |

### 后端环境变量 (Railway)
| 变量名 | 必需 | 说明 |
|--------|------|------|
| `DATABASE_URL` | ✅ | MySQL连接字符串 |
| `JWT_SECRET` | ✅ | JWT密钥 (建议长度>=32字符) |
| `REDIS_URL` | ❌ | Redis连接字符串 (可选) |
| `SERVER_PORT` | ❌ | 服务器端口 (默认: 8080) |
| `SPRING_PROFILES_ACTIVE` | ❌ | Spring配置文件 (默认: prod) |

## API接口概览

### 用户管理 (`UserController.java`)
- `POST /api/users/register` - 用户注册
- `POST /api/users/login` - 用户登录
- `GET /api/users/info` - 获取用户信息

### 商品管理 (`ProductController.java`)
- `GET /api/products` - 获取商品列表
- `GET /api/products/{id}` - 获取商品详情
- `POST /api/products` - 创建商品
- `PUT /api/products/{id}` - 更新商品
- `DELETE /api/products/{id}` - 删除商品
- `GET /api/products/my` - 获取我的商品

### 订单管理 (`OrderController.java`)
- `POST /api/orders` - 创建订单
- `GET /api/orders/{id}` - 获取订单详情
- `GET /api/orders/buyer` - 获取买家订单
- `GET /api/orders/seller` - 获取卖家订单
- `PUT /api/orders/{id}/status` - 更新订单状态
- `DELETE /api/orders/{id}` - 删除订单
- `PUT /api/orders/{id}/confirm` - 确认订单

### 消息管理 (`MessageController.java`)
- `POST /api/messages` - 发送消息
- `GET /api/messages/conversation/{otherUserId}` - 获取对话
- `GET /api/messages/contacts` - 获取联系人
- `GET /api/messages/unread` - 获取未读消息
- `PUT /api/messages/{id}/read` - 标记已读
- `GET /api/messages/unread/count` - 未读消息计数

### 分类管理 (`CategoryController.java`)
- `POST /api/categories` - 创建分类
- `PUT /api/categories/{id}` - 更新分类
- `DELETE /api/categories/{id}` - 删除分类
- `GET /api/categories/{id}` - 获取分类详情
- `GET /api/categories` - 获取所有分类
- `GET /api/categories/roots` - 获取根分类
- `GET /api/categories/parent/{parentId}` - 获取子分类
- `GET /api/categories/tree` - 获取分类树

## 部署步骤

### 1. 准备工作
1. 确保代码已推送到GitHub仓库
2. 注册并登录 [Railway](https://railway.app/) 和 [Vercel](https://vercel.com/)
3. 准备MySQL数据库 (Railway提供或自建)

### 2. 后端部署 (Railway)
1. 在Railway控制台创建新项目
2. 导入GitHub仓库
3. 添加MySQL数据库服务
4. 配置环境变量:
   - `DATABASE_URL`: MySQL连接字符串
   - `JWT_SECRET`: 设置安全的JWT密钥
   - `SPRING_PROFILES_ACTIVE`: `prod`
5. 部署并获取后端URL

### 3. 前端部署 (Vercel)
1. 在Vercel控制台创建新项目
2. 导入GitHub仓库
3. 配置环境变量:
   - `VITE_API_BASE_URL`: 设置为后端部署的URL
4. 部署并获取前端URL

### 4. 验证部署
1. 访问前端URL，确认页面正常加载
2. 测试用户注册和登录功能
3. 测试商品发布和浏览功能
4. 测试订单创建和消息功能

## 常见问题解决方案

### 1. Vercel部署问题
| 问题 | 解决方案 |
|------|----------|
| `vue-cli-service: command not found` | 在vercel.json中明确配置`"framework": "vite"` |
| `vite: command not found` | 使用`npm ci --prefer-offline`替代`npm install` |
| `Could not read package.json` | 确保根目录有package.json文件 |
| SPA 404错误 | 配置路由重写规则: `"rewrites": [{ "source": "/(.*)", "destination": "/" }]` |

### 2. Railway部署问题
| 问题 | 解决方案 |
|------|----------|
| 健康检查失败 | 确保`/actuator/health`端点可访问 |
| 数据库连接失败 | 检查`DATABASE_URL`格式和权限 |
| 端口冲突 | 设置`SERVER_PORT`环境变量 |
| 构建失败 | 检查Maven配置和Java版本 |

### 3. 前后端连接问题
| 问题 | 解决方案 |
|------|----------|
| CORS错误 | 后端已配置CORS，检查前端API地址 |
| API调用失败 | 确认`VITE_API_BASE_URL`设置正确 |
| 认证失败 | 检查JWT令牌和密钥配置 |
